## 🔍 重点测试场景

### 场景1：完整预约流程
```
用户登录 → 浏览服务 → 查看详情 → 选择时间 → 填写信息 → 确认预约 → 查看预约
```
**预期时间**: 3-5分钟  
**关键验证点**:
- 时间段可用状态正确
- 预约信息完整保存
- 预约编号正确生成
- 双方都能看到预约

### 场景2：服务提供者接单
```
发布服务 → 收到预约 → 查看客户信息 → 接受预约 → 联系客户
```
**预期时间**: 3-5分钟  
**关键验证点**:
- 预约通知及时
- 客户信息完整
- 状态更新正确
- 双方状态同步

### 场景3：并发预约测试
```
2个用户同时预约同一时间段（只剩1个名额）
```
**预期时间**: 5分钟  
**关键验证点**:
- 只有1个预约成功
- 另一个收到"已满"提示
- 数据库无重复记录
- 时间段状态正确

## ⚠️ 常见问题排查

### 问题1：后端启动失败
**症状**: 端口占用或数据库连接失败  
**解决**:
```bash
# 检查端口占用
netstat -ano | findstr 8080

# 检查MySQL服务
net start mysql

# 检查数据库配置
# 编辑 application.yml 中的数据库连接信息
```

### 问题2：前端页面空白
**症状**: 浏览器显示空白页  
**解决**:
1. 打开浏览器控制台（F12）
2. 查看Console错误信息
3. 检查Network请求是否404
4. 验证后端API地址配置

### 问题3：预约失败
**症状**: 点击确认预约后失败  
**解决**:
1. 检查时间段配置是否存在
2. 查看后端日志错误信息
3. 验证数据库b_time_slot表数据
4. 检查用户登录状态

### 问题4：图片不显示
**症状**: 服务封面图显示失败  
**解决**:
1. 检查图片路径配置
2. 验证静态资源目录
3. 查看浏览器Network请求
4. 确认图片文件存在
