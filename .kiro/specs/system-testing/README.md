## ⚡ 测试优先级说明

### P0 - 核心功能（必须100%通过）
- 用户登录/注册
- 服务预约流程
- 预约管理
- 管理员基本操作
- 安全性测试

**影响**: 如果P0用例失败，系统无法正常使用

### P1 - 重要功能（建议≥90%通过）
- 搜索与筛选
- 收藏功能
- 个人信息管理
- 数据统计
- 性能测试

**影响**: 如果P1用例失败，影响用户体验

### P2 - 辅助功能（可选）
- 动画效果
- 响应式布局
- 消息提醒

**影响**: 如果P2用例失败，不影响核心功能


### 方法3: 计算完成率
```
完成率 = (已执行用例数 / 总用例数) × 100%
通过率 = (通过用例数 / 已执行用例数) × 100%
```

## 🐛 缺陷管理

### 缺陷严重程度定义

**致命缺陷**:
- 系统崩溃或无法启动
- 数据丢失或损坏
- 安全漏洞

**严重缺陷**:
- 核心功能无法使用
- 数据不一致
- 性能严重下降

**一般缺陷**:
- 功能部分异常
- 界面显示错误
- 提示信息不准确

**轻微缺陷**:
- 界面美观问题
- 文字错误
- 非关键功能小问题

### 缺陷处理流程
```
发现缺陷 → 记录缺陷 → 分配优先级 → 开发修复 → 测试验证 → 关闭缺陷
```

## ❓ 常见问题

### Q1: 测试用例太多，时间不够怎么办？
**A**: 优先执行P0级别用例，确保核心功能正常。P1和P2可以抽查。

### Q2: 发现缺陷后是否继续测试？
**A**: 如果是致命缺陷，停止测试等待修复。如果是一般缺陷，记录后继续测试。

### Q3: 如何判断测试是否充分？
**A**: 
- P0用例100%执行
- 通过率≥90%
- 无致命和严重缺陷
- 核心业务流程完整可用

### Q4: 测试数据如何准备？
**A**: 参考 `design.md` 中的"测试数据准备清单"，或使用 `QUICK_START.md` 中的快速准备方法。

### Q5: 如何进行并发测试？
**A**: 
- 准备多个测试账号
- 使用多个浏览器窗口
- 同时执行相同操作
- 验证数据一致性
