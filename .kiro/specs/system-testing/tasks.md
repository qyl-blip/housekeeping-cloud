## 注意事项

### 测试执行规范
1. 每个测试用例执行前检查前置条件
2. 严格按照测试步骤执行
3. 详细记录实际结果
4. 发现问题立即记录缺陷
5. 截图保存关键步骤

### 缺陷报告规范
1. 缺陷标题简洁明确
2. 复现步骤详细完整
3. 附上截图或日志
4. 标注严重程度和优先级
5. 及时跟踪修复状态

### 测试数据管理
1. 测试前备份数据库
2. 每轮测试后恢复数据
3. 保留关键测试数据
4. 记录数据变更情况

### 时间管理
1. 按模块顺序执行
2. P0用例优先执行
3. 预留回归测试时间
4. 及时更新进度

### 沟通协作
1. 及时反馈测试进度
2. 重大问题立即沟通
3. 定期同步测试结果
4. 协助开发定位问题
