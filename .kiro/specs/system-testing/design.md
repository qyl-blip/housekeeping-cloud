# 系统测试用例设计

## 测试用例结构说明

每个测试用例包含以下信息：
- **用例ID**: 唯一标识符
- **用例名称**: 简短描述
- **优先级**: P0/P1/P2
- **前置条件**: 执行前需要满足的条件
- **测试步骤**: 详细操作步骤
- **预期结果**: 每步的预期输出
- **实际结果**: 执行后填写
- **测试状态**: 通过/失败/阻塞

---

## 模块1: 用户认证与授权

### TC-001: 用户注册功能
**优先级**: P0  
**前置条件**: 
- 系统正常运行
- 数据库连接正常
- 用户未登录状态

**测试步骤**:
1. 访问前台首页 http://localhost:5173
2. 点击右上角"注册"按钮
3. 填写注册信息：
   - 用户名: testuser001
   - 密码: Test@123456
   - 确认密码: Test@123456
   - 手机号: 13800138000
4. 点击"注册"按钮

**预期结果**:
- 步骤1: 页面正常加载，显示首页
- 步骤2: 跳转到注册页面
- 步骤3: 表单正常显示，可输入
- 步骤4: 注册成功，提示"注册成功"，自动跳转到登录页

**数据验证**:
- 数据库b_user表新增一条记录
- 用户名为testuser001
- 密码已加密存储
