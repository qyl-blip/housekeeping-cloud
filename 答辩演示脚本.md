假设一个时间段最多5个人，现在已经有4个人预约了，剩1个名额。如果同时有10个人点预约，怎么保证不会超过5个？"

```java
// housekeeping-api/src/main/java/com/gk/study/service/impl/AppointmentServiceImpl.java
```

**说什么：**
"我用的是数据库的行级锁，也就是 SELECT FOR UPDATE。

你看第373行：
```java
int currentCount = appointmentMapper.countByDateAndSlotWithLock(
```


这样就保证了并发安全，不会超额预约。"

## 答辩常见问题准备

### Q1: 为什么选择微服务架构？

**回答：**
"主要考虑到系统的可扩展性。虽然现在只有两个服务，但微服务架构的好处是：
1. 服务独立部署，互不影响
2. 可以根据业务需求独立扩展某个服务
3. 技术栈可以灵活选择
4. 团队可以并行开发

比如本地推荐服务，如果访问量大，可以单独扩容，不影响主服务。"

---

### Q2: 并发预约是怎么处理的？

**回答：**
"用的是数据库的行级锁，也就是 SELECT FOR UPDATE。

具体流程是：
1. 开启事务
2. 用 SELECT FOR UPDATE 查询当前预约数，这时会锁住这条记录
3. 检查是否超过容量
4. 如果没超过，插入预约记录
5. 提交事务，释放锁

这样可以保证同一时间只有一个事务能修改这条记录，避免超额预约。

我做过并发测试，100个用户同时预约容量为5的时间段，最终只有5个成功，95个失败，数据完全一致。"

---

### Q3: Redis缓存的过期策略是什么？

**回答：**
"我用的是固定过期时间，设置为10分钟。

选择10分钟是平衡了性能和数据新鲜度：
- 太短的话缓存命中率低，起不到优化作用
- 太长的话数据可能不够实时

如果服务信息更新了，会主动删除缓存，下次查询时重新加载。

缓存Key的设计是：local:service:v2:城市名
这样可以按城市分别缓存，还可以通过版本号v2来更新缓存策略。"

---

### Q4: 前端的状态管理是怎么做的？

**回答：**
"用的是Pinia，Vue 3官方推荐的状态管理库。

主要管理这些状态：
- 用户信息（userInfo、token）
- 登录状态（isLogin）
- 购物车数据
- 全局配置

Pinia的好处是：
1. API简单，比Vuex更轻量
2. 支持TypeScript
3. 有持久化插件，可以自动存到localStorage

比如用户登录后，Token会存到Pinia的store，同时持久化到localStorage，刷新页面也不会丢失。"

---

### Q5: 安全性是怎么保证的？

**回答：**
"主要做了这几方面：

1. **密码安全**：MD5加密 + 盐值，不明文存储
2. **Token认证**：每次请求都验证Token，防止未授权访问
3. **权限控制**：拦截器验证用户角色，防止越权操作
4. **SQL注入防护**：用MyBatis-Plus的参数化查询，不拼接SQL
5. **XSS防护**：前端对用户输入做转义
6. **CSRF防护**：Token机制天然防CSRF

如果要上生产环境，还需要：
- HTTPS加密传输
- 接口限流防刷
- 敏感操作二次验证
- 日志审计"

---

### Q6: 如果预约量很大，怎么优化？

**回答：**
"可以从几个方面优化：

1. **数据库层面**：
   - 给 (appointment_date, slot_id) 加组合索引（已经做了）
   - 读写分离，查询走从库
   - 分库分表，按日期分表

2. **缓存层面**：
   - 把时间段可用性缓存到Redis
   - 用Redis的原子操作（INCR/DECR）控制库存
   - 设置短过期时间（1分钟），保证实时性

3. **架构层面**：
   - 预约服务独立出来，单独扩容
   - 用消息队列削峰，异步处理预约
   - 用分布式锁（Redis/Zookeeper）代替数据库锁

4. **业务层面**：
   - 增加预约时间段数量
   - 提高每个时间段容量
   - 引入预约排队机制"

---

### Q7: 为什么用MyBatis-Plus而不是JPA？

**回答：**
"主要考虑到：

1. **灵活性**：MyBatis-Plus可以写复杂SQL，JPA的JPQL有局限
2. **性能**：MyBatis-Plus可以精确控制SQL，优化性能
3. **学习成本**：MyBatis-Plus的API更简单，上手快
