# 家政服务云平台系统 - 毕业设计答辩文档

## 项目概述

### 1.1 项目背景
本系统是一个基于Spring Cloud Alibaba微服务架构的家政服务管理平台，采用前后端分离设计。系统实现了用户管理、服务预约、订单管理、智能推荐等核心功能，为用户提供便捷的家政服务预约体验。

### 1.2 技术架构
- **后端**: Java 8 + Spring Boot 2.5.5 + Spring Cloud Alibaba 2021.1 + MyBatis-Plus 3.5.2
- **前端**: Vue 3.2 + Ant Design Vue 3.2 + TypeScript 4.9 + Vite 4
- **数据库**: MySQL 8.0
- **缓存**: Redis
- **服务治理**: Nacos
- **微服务通信**: OpenFeign

#### 1.2.1 用户注册功能
**实现位置**: `UserController.userRegister()`

**实现原理**:
```java
@RequestMapping(value = "/userRegister", method = RequestMethod.POST)
@Transactional
public APIResponse userRegister(User user) {
    // 1. 参数验证
    if (user == null || !StringUtils.hasText(user.getUsername()) 
        || !StringUtils.hasText(user.getPassword())) {
        return new APIResponse(ResponeCode.FAIL, "参数错误");
    }
    
    // 2. 检查用户名是否已存在
    if (userService.getUserByUserName(user.getUsername()) != null) {
        return new APIResponse(ResponeCode.FAIL, "用户名已存在");
    }
    
    // 3. 验证两次密码是否一致
    if (!user.getPassword().equals(user.getRePassword())) {
        return new APIResponse(ResponeCode.FAIL, "两次密码不一致");
    }
    
    // 4. 密码加密 (MD5 + 盐值)
    String md5Pwd = DigestUtils.md5DigestAsHex(
        (user.getPassword() + salt).getBytes()
    );
    user.setPassword(md5Pwd);
    
    // 5. 生成Token (用于后续认证)
    String token = DigestUtils.md5DigestAsHex(
        (user.getUsername() + salt).getBytes()
    );
    user.setToken(token);
    
    // 6. 设置默认角色和状态
    user.setRole(String.valueOf(User.NormalUser)); // 角色=1(普通用户)
    user.setStatus("0"); // 状态=0(正常)
    user.setCreateTime(String.valueOf(System.currentTimeMillis()));
    
    // 7. 保存到数据库
    userService.createUser(user);
    return new APIResponse(ResponeCode.SUCCESS, "注册成功");
}
```

**关键技术点**:
1. **密码安全**: 使用MD5加密 + 固定盐值("abcd1234")，防止明文存储
2. **Token生成**: 基于用户名+盐值生成唯一Token，用于后续身份验证
3. **事务控制**: @Transactional注解确保数据一致性
4. **参数校验**: 使用StringUtils.hasText()进行非空验证

```java
@Override
public boolean preHandle(HttpServletRequest request, 
                        HttpServletResponse response, 
                        Object handler) throws Exception {
    // 1. 跳过OPTIONS预检请求
    if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
        return true;
    }
    
    // 2. 获取方法上的@Access注解
    HandlerMethod handlerMethod = (HandlerMethod) handler;
    Method method = handlerMethod.getMethod();
    Access access = method.getAnnotation(Access.class);
    
    if (access == null) {
        return true; // 无注解，直接放行
    }
    
    // 3. 从请求头获取TOKEN
    String token = request.getHeader("TOKEN");
    if (token == null || token.trim().isEmpty()) {
        writeResponse(response, 
            new APIResponse(ResponeCode.UNAUTHORIZED, "未登录"));
        return false;
    }
    
    // 4. 根据TOKEN查询用户
    User user = userService.getUserByToken(token);
    if (user == null) {
        writeResponse(response, 
            new APIResponse(ResponeCode.UNAUTHORIZED, "未登录"));
        return false;
    }
    
    // 5. 验证权限级别
    int userRole = Integer.parseInt(user.getRole());
    
    if (access.level() == AccessLevel.ADMIN && userRole < 2) {
        writeResponse(response, 
            new APIResponse(ResponeCode.UNAUTHORIZED, "权限不足"));
        return false;
    }
    
    return true; // 验证通过
}
```

**关键技术点**:
1. **拦截器机制**: 实现HandlerInterceptorAdapter，在请求前验证权限
2. **注解驱动**: 使用@Access注解标记需要权限的接口
3. **TOKEN验证**: 从请求头获取TOKEN，查询用户身份
4. **角色判断**: 根据用户角色(role字段)判断是否有权限访问

#### 2.3.1 获取可用时间段

```java
@Override
public List<TimeSlotDTO> getAvailableSlots(String date, String thingId) {
    // 1. 查询所有启用的时间段配置
    List<TimeSlot> timeSlots = timeSlotMapper.selectAllEnabled();
    
    // 2. 创建结果列表
    List<TimeSlotDTO> result = new ArrayList<>();
    
    // 3. 遍历每个时间段，查询预约数量
    for (TimeSlot slot : timeSlots) {
        // 查询该日期该时间段的已预约数量
        int bookedCount = appointmentMapper.countByDateAndSlot(date, slot.getSlotId());
        
        // 创建DTO对象
        TimeSlotDTO dto = new TimeSlotDTO();
        dto.setSlotId(slot.getSlotId());
        dto.setStartTime(slot.getStartTime());
        dto.setEndTime(slot.getEndTime());
        dto.setBookedCount(bookedCount);
        dto.setMaxCapacity(slot.getMaxCapacity());
        
        // 计算可用状态: 已预约数 < 最大容量
        dto.setAvailable(bookedCount < slot.getMaxCapacity());
        
        result.add(dto);
    }
    
    return result;
}
```

#### 2.3.2 创建预约 (并发控制)
**实现位置**: `AppointmentServiceImpl.createAppointment()`

**实现原理**:
```java
@Override
@Transactional
public void createAppointment(Appointment appointment) {
    // 1. 验证预约日期不能是过去的日期
    LocalDate appointmentDate;
    try {
        appointmentDate = LocalDate.parse(appointment.getAppointmentDate());
    } catch (Exception e) {
        throw new InvalidDateException("日期格式无效");
    }
    
    if (appointmentDate.isBefore(LocalDate.now())) {
        throw new InvalidDateException("不能预约过去的日期");
    }
    
    // 2. 验证时间段ID有效且已启用
    TimeSlot slot = timeSlotMapper.selectBySlotId(appointment.getSlotId());
    if (slot == null || !slot.getEnabled()) {
        throw new InvalidSlotException("时间段无效或已禁用");
    }
    
    // 3. 使用 SELECT FOR UPDATE 锁定记录并查询预约数
    // 这是防止并发超额预约的关键！
    int currentCount = appointmentMapper.countByDateAndSlotWithLock(
        appointment.getAppointmentDate(), 
        appointment.getSlotId()
    );
    
    // 4. 检查预约数是否已达到最大容量
    if (currentCount >= slot.getMaxCapacity()) {
        throw new AppointmentConflictException("该时间段已被预约，请选择其他时间");
    }
    
    // 5. 生成唯一的预约编号
    // 格式: APT + yyyyMMdd + 6位序列号
    String appointmentNumber = AppointmentNumberGenerator.generate();
    appointment.setAppointmentNumber(appointmentNumber);
    
    // 6. 插入预约记录到数据库
    appointmentMapper.insert(appointment);
}
```

**并发控制实现** (`AppointmentMapper.xml`):
```xml
<!-- 使用 FOR UPDATE 实现行级锁 -->
<select id="countByDateAndSlotWithLock" resultType="int">
    SELECT COUNT(*)
    FROM b_appointment
    WHERE appointment_date = #{date}
      AND slot_id = #{slotId}
      AND status = '0'
    FOR UPDATE  -- 关键：锁定查询结果，防止并发问题
</select>
```

**预约编号生成器** (`AppointmentNumberGenerator.java`):
```java
public class AppointmentNumberGenerator {
    private static final AtomicInteger sequence = new AtomicInteger(0);
    
    public static String generate() {
        // 格式: APT + yyyyMMdd + 6位序列号
        String date = LocalDate.now().format(
            DateTimeFormatter.ofPattern("yyyyMMdd")
        );
        int seq = sequence.incrementAndGet() % 1000000;
        return String.format("APT%s%06d", date, seq);
    }
}
```

**关键技术点**:
1. **并发控制**: 使用SELECT FOR UPDATE实现行级锁，防止超额预约
   - 事务A查询并锁定记录
   - 事务B等待事务A完成
   - 确保同一时间段不会被超额预约
2. **日期验证**: 使用LocalDate.parse()验证日期格式，防止预约过去的日期
3. **事务管理**: @Transactional确保查询和插入在同一事务中
4. **唯一编号**: 使用日期+序列号生成唯一预约编号

```java
@Override
public List<Thing> getThingList(String keyword, String sort, String c, String tag) {
    QueryWrapper<Thing> queryWrapper = new QueryWrapper<>();
    
    // 1. 关键词搜索(模糊查询)
    queryWrapper.like(StringUtils.isNotBlank(keyword), "title", keyword);
    
    // 2. 排序方式
    if (StringUtils.isNotBlank(sort)) {
        if (sort.equals("recent")) {
            // 按创建时间倒序
            queryWrapper.orderBy(true, false, "create_time");
        } else if (sort.equals("hot") || sort.equals("recommend")) {
            // 按浏览量倒序
            queryWrapper.orderBy(true, false, "pv");
        }
    } else {
        queryWrapper.orderBy(true, false, "create_time");
    }
    
    // 3. 分类筛选
    if (StringUtils.isNotBlank(c) && !c.equals("-1")) {
        queryWrapper.eq(true, "classification_id", c);
    }
    
    // 4. 执行查询
    List<Thing> things = mapper.selectList(queryWrapper);
    
    // 5. 标签筛选(需要关联查询)
    if (StringUtils.isNotBlank(tag)) {
        List<Thing> tThings = new ArrayList<>();
        QueryWrapper<ThingTag> thingTagQueryWrapper = new QueryWrapper<>();
        thingTagQueryWrapper.eq("tag_id", tag);
        List<ThingTag> thingTagList = thingTagMapper.selectList(thingTagQueryWrapper);
        
        // 筛选出包含指定标签的服务
        for (Thing thing : things) {
            for (ThingTag thingTag : thingTagList) {
                if (thing.getId().equals(thingTag.getThingId())) {
                    tThings.add(thing);
                }
            }
        }
        things = tThings;
    }
    
    // 6. 为每个服务加载标签列表
    for (Thing thing : things) {
        QueryWrapper<ThingTag> thingTagQueryWrapper = new QueryWrapper<>();
        thingTagQueryWrapper.lambda().eq(ThingTag::getThingId, thing.getId());
        List<ThingTag> thingTags = thingTagMapper.selectList(thingTagQueryWrapper);
        List<Long> tags = thingTags.stream()
            .map(ThingTag::getTagId)
            .collect(Collectors.toList());
        thing.setTags(tags);
    }
    
    return things;
}
```

**关键技术点**:
1. **MyBatis-Plus条件构造器**: 使用QueryWrapper实现动态SQL
2. **模糊查询**: like()方法实现标题搜索
3. **多条件组合**: 支持关键词、排序、分类、标签的组合查询
4. **关联查询**: 通过thing_tag表实现标签筛选
5. **Lambda表达式**: 使用Stream API处理标签列表

#### 5.2.3 Redis缓存优化
**实现位置**: `LocalService.getServicesByCity()`

**实现原理**:
```java
@Service
public class LocalService {
    @Autowired
    private RedisTemplate redisTemplate;
    
    public List<Thing> getServicesByCity(String city) {
        // 1. 定义缓存Key
        String cacheKey = "local:service:v2:" + city;
        
        // 2. 先查Redis缓存
        if (redisTemplate.hasKey(cacheKey)) {
            System.out.println(">>> 命中缓存，直接返回Redis数据");
            return (List<Thing>) redisTemplate.opsForValue().get(cacheKey);
        }
        
        // 3. 缓存未命中，查询数据库
        System.out.println(">>> 缓存未命中，查询数据库...");
        QueryWrapper<Thing> query = new QueryWrapper<>();
        query.eq("city", city);
        List<Thing> list = thingMapper.selectList(query);
        
        // 4. 写入Redis，设置10分钟过期
        if (list != null && !list.isEmpty()) {
            redisTemplate.opsForValue().set(cacheKey, list, 10, TimeUnit.MINUTES);
        }
        
        return list;
    }
}
```

**关键技术点**:
1. **缓存策略**: Cache-Aside模式(旁路缓存)
   - 先查缓存
   - 缓存未命中查数据库
   - 查询结果写入缓存
2. **过期时间**: 设置10分钟过期，平衡数据新鲜度和性能
3. **缓存Key设计**: 使用"业务:版本:参数"格式，便于管理

```java
     */
    public void calculateScore(List<Thing> things, double userLat, double userLng) {
        for (Thing t : things) {
            // 1. 距离因子(距离越近分数越高)
            double distance = 0;
            if (t.getLatitude() != null && t.getLongitude() != null) {
                // 计算用户位置与服务位置的距离
                // distance = LocationUtils.getDistance(userLat, userLng, 
                //                                      t.getLatitude(), t.getLongitude());
            }
            
            // 2. 热度因子(浏览量越高分数越高)
            double heatScore = t.getVisitCount() * 0.5; // 权重0.5
            
            // 3. 价格因子(价格越低分数越高)
            double priceScore = (1000 - t.getPrice()) * 0.3; // 权重0.3

            double finalScore = heatScore + priceScore;

    }
}
```
